# 2.5.5 HTTP/3

TCP 위에서 돌아가는 HTTP/2와는 달리, HTTP/3은 QUIC라는 새로운 전송 프로토콜 위에서 실행된다.

```
🥸 QUIC 란 ?
사람들이 하루종일 이동하면서 끊임없이 하나의 네트워크에서 다른 네트워크로 전환하는 스마트폰, 즉 모바일 중심의 인터넷 사용을 위해 설계되었다.
```

QUIC를 사용한다는 것은 HTTP/3이 전송 제어 프로토콜(TCP)이 아닌, 사용자 데이터그램 프로토콜(UDP)에 의존한다는 것이다.

UDP로 전환하면 온라인 검색 시 연결 속도가 빨라지고 사용자 경험이 향상된다.

> 이건 tcp에서 사용하던 3-way handshake 과정을 생략하기 때문이겠죠?
> 한번 요청 및 응답이 왔다갔다 하면 바로 본통신을 시작 가능한것임.. !

### 새 버전의 HTTP가 필요한 이유 ?

QUIC는 HTTP/2의 가장 큰 단점 몇 가지를 해결하고 있다.

- 스마트폰이 wifi에서 셀룰러 데이터로 전환될 때 성능이 저하되는 문제를 해결하였다.
- 패킷 손실의 영향이 감소되었다. 순방향 오류 수정 매커니즘 (FEC; forward error correction)이 적용되어, 전송한 패킷이 손실되었다면 수신 측에서 에러를 검출하고 수정하는 것이다.
